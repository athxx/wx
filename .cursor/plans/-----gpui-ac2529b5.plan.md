# 微信界面优化改进计划

## 改进目标

将当前的基础微信界面改进到更接近真实微信/企业微信的效果，包括视觉样式、群组功能和交互细节。

## 实施步骤

### 步骤 0: 修复关键问题（优先）

**问题清单**:

1. ❌ 左侧工具栏图标不显示、颜色不对
2. ❌ 窗口顶部缺少最小化、最大化、关闭按钮
3. ❌ 发送按钮颜色不对
4. ❌ 会话列表和聊天区域应该可以调整大小

**修复方案**:

**0.1 添加窗口标题栏**

- 参考 `title_bar.rs`，使用 `TitleBar::new()` 组件
- 在 `main.rs` 中集成标题栏到主窗口
- 标题栏包含最小化、最大化、关闭按钮

**0.2 实现可调整大小的布局**

- 参考 `resizable_story.rs` 使用 `h_resizable` 和 `resizable_panel`
- 会话列表设置为可调整大小（200px-400px）
- 聊天区域自动填充剩余空间
- 工具栏保持固定宽度

**0.3 修复图标显示问题**

- 检查图标是否正确导入和使用
- 确保 Button 的 ghost 样式正确应用
- 修复图标颜色（未选中 #999999，选中 #07c160）

**0.4 修复发送按钮颜色**

- 检查 primary 按钮主题色
- 应该是绿色 (#07c160) 或微信蓝色

### 步骤 1: 改进输入区域样式

**文件**: `src/components/chat_area.rs`

参考 `D:\rust_test\gpui-component\crates\story\src\input_story.rs` 的 custom_input 实现：

- 使用 `TextInput::new(&self.input_state).appearance(false)` 移除默认样式
- 在外层容器自定义样式（边框、背景、内边距）
- 调整输入框高度，使其更符合微信风格
- 优化发送按钮位置和样式
- 简化工具栏按钮样式

**预期效果**:

- 输入框无明显边框，融入白色背景
- 初始高度适中，支持多行但不过高

### 步骤 2: 优化工具栏图标和样式

**文件**: `src/components/toolbar.rs`

- 调整工具栏宽度（从 64px 改为 52-54px）

**预期效果**:

- 工具栏更窄，节省空间
- 图标大小适中，视觉平衡
- 选中状态更清晰

### 步骤 3: 添加群组数据模型支持

**文件**: `src/models.rs`

扩展数据模型以支持群组：

- 为 `Contact` 添加 `is_group` 字段
- 添加 `members` 字段存储群成员（用于头像显示）
- 添加 `member_count` 字段显示成员总数
- 修改 `Message` 结构，在群聊中显示发送者名称

**数据结构**:

```rust
pub struct Contact {
    pub id: String,
    pub name: String,
    pub is_group: bool,              // 新增
    pub member_count: Option<u32>,   // 新增
    pub avatar_members: Vec<String>, // 新增：用于显示的成员名称
    // ... 其他字段
}
```

### 步骤 4: 实现群组头像组件

**文件**: 新建 `src/components/group_avatar.rs`

创建群组头像组件，显示 2x2 网格：

- 接收最多 4 个成员名称
- 将 4 个小头像排列成 2x2 网格
- 每个小头像使用 `Avatar` 组件，但尺寸减半
- 整体保持与单人头像相同的外部尺寸

**组件结构**:

```rust
pub struct GroupAvatar {
    members: Vec<String>,  // 最多4个

}
```

## 技术要点

1. **appearance(false)**: 移除 TextInput 默认样式
2. **GroupAvatar**: 使用 Grid 或 Flex 布局实现 2x2 头像网格
3. **条件渲染**: 使用 `.when()` 区分群组和个人
4. **Badge 组件**: 显示未读消息数
5. **颜色方案**: 使用 rgb() 宏定义精确颜色
